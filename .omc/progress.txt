### Key Implementation Details

1. **Resource Path Structure**
   - Desktop: Uses FileSystemAdapter.getBasePath() to get vault root
   - Mobile: Falls back to vault-internal storage
   - Resources stored at: {vault-parent}/resources/{type}/{md-relative-path}/

2. **File Type Organization**
   - images: jpg, jpeg, png, gif, webp, svg, bmp, ico
   - audio: mp3, wav, ogg, flac, m4a
   - videos: mp4, webm, mov, avi, mkv
   - documents: pdf, doc, docx, xls, xlsx, ppt, pptx
   - archives: zip, rar, 7z, tar, gz
   - code: js, ts, py, java, cpp, c, go, rs, html, css, json, xml, yaml, yml, sql, sh, bat, ps1
   - files: all others

3. **Duplicate Handling**
   - Automatically appends numeric suffix
   - Example: image.png → image-1.png → image-2.png

4. **Link Formats Supported**
   - WikiLinks: ![[resources/images/path/file.png]]
   - Markdown: ![alt](resources/images/path/file.png)
